# 热词命中率统计 (PR-A)

## 概述

本特性让你看到自己配置的 hotword 列表中"哪些真在帮助识别、哪些是死词"。

打开 `设置 -> Transcriptionmodels -> 热词命中率` 面板，可以看到每个 hotword 的三项数据：

- hotword：词条名称
- 命中次数：转写后产物中识别出该词的累计次数
- 最后命中时间：最近一次识别到该词的时间戳

## 规则

- **匹配规则**：全字精确 + 大小写不敏感。"OpenAI" 命中 "openai"、命中 "OPENAI"，但不会命中 "OpenAIs"
- **保留期**：30 天滚动清理。一个 hotword 在最近 30 天内从未命中，会从统计中移除
- **去前缀**：支持的 `!前缀` 保护语法按原词参与统计（如 `!OpenAI` 与 `OpenAI` 合并为一个条目）

## 不在统计范围

- 命中率百分比（命中数 / 总词数）：本 PR 不提供
- 跨设备/多端同步：本数据仅在本机存储
- 自动热词重排序：本 PR 不提供

## 数据存储位置

- 表：`hotword_hit_stats`
- 行数：与你的 hotword 列表同阶（每个 hotword 最多 1 行）

## 性能影响

- 每次 ASR segment 落地后写入 < 0.5ms
- 1 小时会议（~300 段）累计开销约 1.5 秒
