## Task 3: PR-27 zh-CN 翻译质量优化

### 3.1 准备

- 备份当前 zh-CN:用 `git diff` 对比前后变化。
- 列出所有 zh-CN JSON 文件:`frontend/locales/zh-CN/*.json`
- 按 namespace 分:common / recording / transcript / summary / settings / errors / sidebar

### 3.2 优化策略

按用户偏好「简洁优先」+ 「自然度」+ 「术语统一」:

- **去除 AI 痕迹**:不出现「轻松」「完美」「强大」「智能」等修饰词
- **简化句式**:中文短句优先,避免长句
- **技术术语**:
  - Whisper / Parakeet / Ollama / OpenRouter / Claude / Groq / BlackHole 保留英文(品牌名/模型名)
  - meeting / transcript / summary / recording 等技术行为词用统一译法
- **占位符**:`{name}` `{version}` 等不变
- **标点**:中文全角
- **连接词**:`和` 而非 `&`;`或` 而非 `/`(技术或路径才用 `/`)

### 3.3 执行

按 namespace 顺序(因 settings 依赖性最强,先做):

1. `zh-CN/errors.json` — 短,先热身
2. `zh-CN/common.json` — 通用短语,影响最广
3. `zh-CN/sidebar.json` — 简单
4. `zh-CN/recording.json` — 录音
5. `zh-CN/transcript.json` — 转录
6. `zh-CN/summary.json` — 摘要
7. `zh-CN/settings.json` — 最大,放最后

每完成 1 个 namespace 立即:
```bash
pnpm run check:i18n && pnpm test:i18n
```
不通过则回滚当前 namespace。
