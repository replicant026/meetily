# Wave 15: Chinese Hot-word Injection (PR-45c)

> **Base branch:** feature/stability-wave8
> **Parent:** Wave 15 PR-45a (Chinese model evaluation), shipped

## Background

Whisper recognizes common Chinese reasonably well but struggles with:
- Company / project names ("Meetily", "OKR", "K8s")
- Technical jargon in mixed-language meetings
- Speaker names that appear for the first time

whisper.cpp exposes `params.set_initial_prompt(text)` which biases the
decoder toward known vocabulary. PR-45c wires this into the meetily
transcription pipeline so future UI work (hot-word editor) can pass
user-configured prompts.

## Goals

- `transcribe_audio_with_confidence` accepts `initial_prompt: Option<String>`.
- When supplied, the prompt is forwarded to `params.set_initial_prompt`.
- All existing call sites pass `None` — zero behavior change for current users.
- Document usage in `docs/asr_benchmark_zh.md` so future UI PRs have a reference.

## Non-Goals

- **UI hot-word editor.** Tracked as PR-45c-ui (separate, larger PR).
- **Per-meeting prompt persistence.** Belongs in the meeting config table
  and is a much larger change touching the database schema.
- **Automatic prompt generation from meeting title / participants.** Out of
  scope; would require NLP heuristics.
- **Provider trait changes.** `TranscriptionProvider::transcribe(audio, language)`
  is left untouched to keep the abstraction stable. UI work that needs
  prompt access should add it at the provider layer in a follow-up PR.

## Design

```
transcribe_audio_with_confidence(audio, language, initial_prompt)
  └─ if let Some(prompt) = initial_prompt where !prompt.trim().is_empty():
         params.set_initial_prompt(prompt)
```

The empty-prompt check ensures callers can safely pass `Some("".to_string())`
without triggering whisper.cpp internal state confusion.

## Risks

| Risk | Mitigation |
|---|---|
| Bad prompt introduces hallucinations | Empty/whitespace check skips the call; documented in PR |
| Prompt too long (>224 tokens for large-v3) | whisper.cpp silently truncates; documented limit |
| Provider trait out of sync with engine API | Trait intentionally left unchanged; UI PR will add prompt at provider layer |

## Sandbox caveat

This environment lacks `cargo build`. The Rust changes were AST-verified:
- `Option<String>` parameter added at correct position (last positional arg)
- All 4 call sites updated to pass `None` (positional, no other changes)
- `params.set_initial_prompt` is a standard whisper-rs API (verified against
  whisper-rs 0.13+ FullParams docs).

A full `cargo build` + manual recording test with a real hot-word prompt
is recommended before merging upstream.
