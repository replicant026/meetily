# Wave 9: 6-locale expansion (ja-JP + ko-KR)

> **For agentic workers:** REQUIRED SUB-KILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.
> **基于分支:** feature/stability-wave8

## 背景

Wave 1-8 完成 zh-CN + en-US + en-GB + zh-TW 四语 i18n 体系（glossary 81 术语 + CI 硬化），同时 Wave 8 完成 4 个稳定性加固 PR。

本 wave 进一步把 locale 矩阵从 **4 语言扩到 6 语言**，命中用户硬约束"增加用户基数"：

- **ja-JP** — 日语市场（日本 + 东亚 IT 圈），最高用户基数扩展
- **ko-KR** — 韩语市场（韩国 + 东亚 IT 圈），紧追日语

复用 Wave 7 已验证的 i18n 流水线（config.ts + request.ts + check-i18n + glossary）。UI 切换器自动渲染新增选项。不引入新依赖。

## Goals

- 沿用 Wave 1-7 i18n 体系，无破坏性变更
- 翻译质量对齐 `docs/i18n/glossary.md` 81 术语（技术术语保留英文）
- 日语遵循日本 IT 圈用语习惯；韩语遵循韩国 IT 圈用语习惯
- 不动 Rust 后端；纯前端 i18n

## Architecture

### A. PR-35 多 locale 基础设施

1. `config.ts`:
   - `LOCALES = ["en-US", "en-GB", "zh-CN", "zh-TW", "ja-JP", "ko-KR"] as const`
   - `resolveLocaleFromOs`: `ja-*` / `ja` → `ja-JP`；`ko-*` / `ko` → `ko-KR`
   - `DEFAULT_LOCALE` 保持 `"en-US"`

2. `request.ts`:
   - 14 个新 import（7 ja-JP + 7 ko-KR）
   - `Messages` type 扩 2 个 namespace
   - `MESSAGES` 字典加 `ja-JP` / `ko-KR` 入口，初始 = en-US 镜像

3. `check-i18n.ts`: 已经用 LOCALES，零改动

4. `frontend/locales/ja-JP/{common,recording,transcript,summary,settings,errors,sidebar}.json`:
   - 初始 = en-US 全量复制（check-i18n 必须立刻通过）

5. `frontend/locales/ko-KR/{common,recording,transcript,summary,settings,errors,sidebar}.json`:
   - 初始 = en-US 全量复制

6. `settings.json` 加 2 个 key（两侧都加）：
   - `preference.locale_name_ja_jp = "日本語"` / `"日本語"`
   - `preference.locale_name_ko_kr = "한국어"` / `"한국어"`

7. `UiLanguagePicker.tsx` 无需改 — 已经在迭代 `LOCALES`

### B. PR-36 ja-JP 全量翻译

每个文件按以下策略：
- UI 文案 → 日语
- 技术术语保留英文（与 Wave 7 zh-TW 策略一致）：Whisper / Parakeet / Ollama / OpenRouter / Claude / Groq / BlackHole / Meetily / Beta / JSON / Tauri / API 等
- 敬体（です/ます）保持一致
- 复数/计数：用日语固有表达（如 `~件` `~個`）
- 日语空格习惯：technical 英文前后保留 1 个半角空格

### C. PR-37 ko-KR 全量翻译

每个文件按以下策略：
- UI 文案 → 韩语
- 技术术语保留英文（同 ja-JP）
- 敬体（합니다 体）保持一致
- 韩语空格习惯：technical 英文前后保留 1 个半角空格
## Cross-cutting constraints

1. 不引入新依赖
2. 不动 Rust 后端
3. 不动 next.config / tailwind.config / build 配置
4. 每 PR 一个 commit（沿用 Wave 1-8 pattern）
5. 三道闸:check:i18n + test:i18n + build 11 static pages
6. 现有 7 namespace 不变（common / recording / transcript / summary / settings / errors / sidebar）
7. 严格按 `LOCALES` 数组顺序（影响 `UiLanguagePicker` 下拉顺序）
8. 翻译术语例外:Whisper / Parakeet / Ollama / OpenRouter / Claude / Groq / BlackHole / Meetily / Beta / JSON / Tauri / API 等保留英文

## Risk & Mitigation

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| ja-JP 翻译质量不达标(机器翻译痕迹) | M | M | 人工翻译 + glossary 对照;敬体统一 |
