# Wave 6: i18n hardening + zh-CN translation quality

> **For agentic workers:** REQUIRED SUB-KILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

## Goals

Wave 1–5 完成 zh-CN 端到端接入,但工程质量与翻译质量存在遗留问题。本 wave 收尾:

1. **质量 (A)** — 修 Wave 1 glossary 编码问题;扩展到 80+ 术语覆盖 Wave 2–5 引入的词条;对全 ~200 keys 做 zh-CN 自然度优化,对照 glossary 统一术语。
2. **工程化 (C)** — 升级 `i18n-check.yml`(扩 branches + 加 `pnpm build` step);PR template 加 i18n 勾选;`docs/i18n/README.md` 加质量审查流程。

## Architecture

### A. Translation quality (PR-27)

- 不动 en-US JSON(英文是 source of truth)。
- 重新写 zh-CN JSON 的字符串值,遵循原则:
  - **自然度**:去除翻译腔,符合中国大陆用户日常表达
  - **术语统一**:每条术语以 glossary.md `approved` 为准
  - **简洁**:技术术语可在括号内补英文(如:转录(transcript))只对不常见词,常见词不补
  - **不啰嗦**:避免 AI 标志词(「轻松」「完美」「强大」),保持克制
  - **标点**:中文用全角(。,;:!?);引号用「」或直接""
  - **占位符**:保持 `{name}` / `{version}` 等不变
- 逐 namespace 优化,每 namespace 完成后立即 check:i18n + test:i18n + build。
- 一次性集中提交(按用户偏好「简洁优先」),不分小 PR。

### C. Engineering hardening (PR-26)

- `glossary.md`:从 20 术语扩到 80+,覆盖 Wave 2–5 引入的所有 UI 词;统一 Markdown 表格对齐;修复 UTF-8 编码显示。
- `i18n-check.yml`:branches 从 `[main, devtest, feature/i18n-wave1, feature/i18n-design-spec]` 扩到覆盖所有 active wave 分支(目前:wave1-wave6 + design-spec);加 `pnpm build` step;cache 优化。
- `PR template`:加 i18n 勾选项(「已 check:i18n 通过」「已术语对照 glossary」「无新增硬编码英文」),仅当 PR 涉及 `frontend/locales/**` 时勾选。
- `README.md`:补充「翻译质量审查」段落,要求 PR 含 zh-CN 改动时附 1 位中文母语 reviewer。

## Cross-cutting constraints

1. 不引入新 namespace(沿用 7 个)。
2. 不动 en-US(en-US 是 source of truth,本 wave 不动)。
3. 不进 `next.config` / `tailwind.config` / 任何 build 配置。
4. JSON 文件 no BOM,UTF-8,2-space 缩进。
5. 每 PR 一个 commit(沿用 Wave 1–5 pattern)。
6. 3 道闸:check:i18n + test:i18n 15-15 + build 11-11。
