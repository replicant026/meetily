## Goal

Promote everything currently sitting under `[Unreleased]` in
`CHANGELOG.md` (Wave 21-24 cumulative) to a real, versioned release
`v0.5.0`. The only existing git tag is `v0.4.0`; the `v0.18.0` /
`v0.19.0` headings in CHANGELOG were forward-looking and never had a
matching tag. This PR closes that gap.

## Semver rationale

- Wave 24 (PR-42-iv-b) introduces a **wire-format breaking change**:
  `transcript-postprocess-failed` payload `error` field flips from
  `String` to `{code, message}`. The frontend hook is updated in the
  same commit, so end users are unaffected, but the wire contract
  changed. That alone justifies a minor bump (`0.4.0` → `0.5.0`).
- No deprecations, no removals beyond the wire-format flip.
- No DB schema migrations outside what was already landed in PR-A
  (Wave 22) and earlier.
