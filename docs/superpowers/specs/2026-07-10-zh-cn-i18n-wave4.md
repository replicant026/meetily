# Wave 4: 残留 i18n polish spec

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

## Goals

完成 Wave 1~3 之后的英文残留，使 zh-CN 模式下整个录制/转写/总结 UI 完全本地化。扩展现有 locale 命名空间（`recording` / `transcript` / `summary` / `sidebar`）的 keys，不新建 namespace。

## Scope

| PR  | Component(s)                    | Keys | 范畴 |
|-----|---------------------------------|------|------|
| 18  | RecordingControls (remaining)   | 14   | 状态语、4 套 DeviceError、close alert、3 个 pause-stop-resume 提示、3 个 alert 错误、validating |
| 19  | TranscriptRecovery (remaining)  | 14   | dialog description、preview 状态（loading/empty/select-no-preview）、3 个按钮（Cancel/Delete/Recover）+ 处理态、3 个 native alert/confirm、Audio/No audio、Started |
| 20  | SummaryPanel + SGBG + RetranscribeDialog + MeetingDetails TranscriptPanel | ~17 | SummaryPanel autoSubtitle / aria-label / title / dominant hint / 2 个 toast description、SGBG 9 个 model toast、RetranscribeDialog 2 个 toast、MD-TranscriptPanel placeholder |
| 21  | Sidebar a11y + (optional sweep) | 2+   | Sidebar 2 个 aria-label 遗漏（Edit meeting title / Delete meeting）。其他可见组件的扫尾留 Wave 5 |

合计约 ~47 keys。

## Architecture

- 严格沿用 Wave 3 pattern：每个 component 加 `useTranslations(<namespace>)` hook、`<string>` 替换成 `{t("namespace.key")}`、JSON 两侧同步加 key。
- 不新增 namespace、不引入新依赖。
- StatusOverlays 已迁移；不在 Wave 4 范围。

## Cross-cutting constraints

1. **`check:i18n` 必须通过**：所有新增 key 在 en-US 与 zh-CN 两侧同步添加，缺一即 build fail。
2. **`pnpm test:i18n` 必须 15/15 通过**。
3. **`pnpm build` 必须无 `MISSING_MESSAGE`**：一旦新增 namespace key 没注册到 `MESSAGES` 字典，next-intl 渲染时会失败。
4. **每 PR 一个 commit，独立 reviewable**。
5. **JSON 文件无 BOM**：用 `[System.Text.UTF8Encoding]::new($false)` 或 node fs.writeFileSync(`utf-8`)。

## Out of scope (deferred)

- 日期/时间格式化本地化（`date-fns` formatDistanceToNow、`toLocaleString` 等）：需 date-fns locale 注册，单 PR 风险太大，留 Wave 5。
- Notes / Settings / PermissionWarning / ModelSelector 等其他组件的小段英文留 Wave 5 或后续。
- 复杂数字/复数（`{n} meeting(s)`）：现状保留英文 fallback。

## Acceptance criteria
