## 1. Background

Wave 1 shipped the locale JSON baseline (12 files, 6 namespaces). Wave 2 wired up persistence and a UI language picker. Only 6 strings in `page.tsx` were actually migrated in Wave 1.

Wave 3 migrates the remaining hardcoded English strings in the always-visible or frequently-visible components: sidebar navigation, recording controls, transcript panel + recovery dialog, and summary panel.

Total scope: **~60 strings across 5 files**. No new dependencies; no architectural changes; pure `useTranslations()` migration.

## 2. Goals & Non-Goals

**Goals:**
- Migrate Sidebar nav labels, status badges, search placeholder, edit-title dialog.
- Migrate RecordingControls: start/stop labels, processing badge, status messages.
- Migrate TranscriptPanel: copy/language buttons, titles.
- Migrate TranscriptRecovery dialog: title, descriptions, action buttons.
- Migrate SummaryPanel: section headers, status messages, action buttons.
- All keys use existing JSON namespaces (`sidebar`, `recording`, `transcript`, `summary`); add a new `sidebar.json` namespace.

**Non-Goals (deferred to later waves):**
- Migrating Settings page tabs, model download dialogs, About page, onboarding steps (lower visibility).
- Migrating modal errors with description strings (e.g. `toast.error("X", { description: "..." })` — description often contains dynamic content; deferred to Wave 4).
- Migrating ARIA labels and tooltip titles for accessibility (Wave 5+).

## 3. Architecture

### 3.1 New namespace: `sidebar.json`

Sidebar has its own concerns (navigation, meeting list, search) that don't fit cleanly into `common`. Create `frontend/locales/{en-US,zh-CN}/sidebar.json` with three top-level keys:
- `nav` — section titles (Home, Meetings, Notes, Settings, Import)
- `meeting_list` — search placeholder, empty states, edit title dialog
- `status` — recording indicator, sync labels

### 3.2 Extend existing namespaces

- `recording.json`: add `controls.processing`, `controls.start_short`, `controls.stop_short` (used in tooltips)
- `transcript.json`: add `panel.copy`, `panel.copy_title`, `panel.language`, `panel.language_title`, `recovery.preview_title`, `recovery.preview_empty`, `recovery.delete_confirm`
- `summary.json`: add `panel.title`, `panel.sections.{key_points,action_items,decisions,main_topics,full_summary}`, `panel.actions.{regenerate,edit,copy}`, `status.{generating,saved_locally,failed_to_save_lang}`

### 3.3 Migration pattern (already established in Wave 1)

For each component:
1. Add `import { useTranslations } from "next-intl";`
2. `const t = useTranslations("<namespace>");` at the top of the component body
3. Replace hardcoded strings with `t("key.path")`
4. For namespace-bound strings (e.g. summary section keys), use `t(\`panel.sections.\${key}\`)`
5. No changes to props, event handlers, imports beyond what's needed

### 3.4 No new tests required for this wave

Each migration is mechanical: `const X = "Foo"` becomes `const X = t("foo")`. Visual correctness is verified manually (tauri:dev). The existing `check:i18n` enforces key sync between locales.

If a component already has a test file (none do in target scope), extend it. Otherwise skip.
