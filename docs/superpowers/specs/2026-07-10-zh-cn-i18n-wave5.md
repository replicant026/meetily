# Wave 5: Settings/Modal/Notes/Update/Analytics polish + date-fns locale

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

## Goals

完成 Wave 1~4 之后剩余的全部英文 UI 残留：Settings 全 tab / PermissionWarning / About / EmptyState / Modal / Notes / Update / Analytics 提示 + date-fns locale 与 `toLocaleString` 时间格式本地化，使 zh-CN 模式 100% 无英文。

| 22  | Settings shell + tabs header | `app/settings/page.tsx`, `components/SettingTabs.tsx` | ~10 |

合计约 ~125 keys + 1 个新模块（date locale）。

## Architecture

- 沿用 Wave 3/4 pattern：每个 component 加 `useTranslations(<namespace>)`，`<string>` → `{t("namespace.key")}`，JSON 双侧同步。
- 扩展现有 `settings` / `common` / `transcript` / `recording` / `summary` / `errors` namespace，不新建。
- 新建 `src/lib/date-locale.tsx`（或内联）做基于 `useLocale()` 的 date-fns/zh-CN locale 动态选择；在 `formatDistanceToNow`/`toLocale*String` 调用处接入。

## Cross-cutting constraints

1. `check:i18n` 通过：新增 keys 双侧同步；Date locale 失败回退 en-US，确保不影响 test。
2. `pnpm test:i18n` 必须 15/15 通过（不引入新 namespace → 测试文件不动）。
3. `pnpm build` 无 `MISSING_MESSAGE`。
4. 每 PR 一个 commit，独立 reviewable。
5. JSON 文件无 BOM：`utf-8` 编码或 `[System.Text.UTF8Encoding]::new($false)`。
6. **不进 `next.config` 等 build 配置**：date locale 注册走单独 import。
