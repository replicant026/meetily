# Wave 7: multi-locale expansion (en-GB + zh-TW)

> **For agentic workers:** REQUIRED SUB-KILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

## Goals

Wave 1–6 完成 zh-CN 端到端 i18n、术语统一、CI 硬化。本 wave 把 locale 矩阵从 2 语言扩到 4 语言,直击「增加用户基数」目标:

- **en-GB** — 英国英语市场(国际企业、欧盟用户),英式拼写精修
- **zh-TW** — 繁体中文市场(台湾、香港、澳门、海外华人),最大化繁中用户覆盖

UI 切换器自动渲染新增选项(check-i18n 自动泛化);不引入新依赖。

## Architecture

### A. Infrastructure (PR-28)

1. `config.ts`:
   - `LOCALES = ["en-US", "en-GB", "zh-CN", "zh-TW"]`
   - `DEFAULT_LOCALE` 保持 `"en-US"`
   - `resolveLocaleFromOs`: `zh-*` → 优先 `zh-TW` 再 `zh-CN`(`zh-Hant`/`zh-HK` → `zh-TW`;`zh-Hans`/`zh-CN` → `zh-CN`);`en-GB` 优先匹配,否则 `en-*` → `en-US`

2. `request.ts`:
   - import 4 套 JSON(共 14 个新 import)
   - `Messages` type 同步扩(2 个新 namespace 类型)
   - `MESSAGES` 字典加 `en-GB` / `zh-TW` 入口,内容初始 = en-US 镜像

3. `check-i18n.ts`:
   - 删除硬编码 `const LOCALES = ["en-US", "zh-CN"]`
   - `import { LOCALES } from "../i18n/config"`
   - 行为对所有 LOCALES 一视同仁

4. `frontend/locales/en-GB/{common,recording,transcript,summary,settings,errors,sidebar}.json`:
   - 初始内容 = `en-US` 对应文件全量复制(check-i18n 必须立刻通过)

5. `frontend/locales/zh-TW/{common,recording,transcript,summary,settings,errors,sidebar}.json`:
   - 初始内容 = `en-US` 对应文件全量复制(不复制 zh-CN,避免简繁混用;后续 PR-30 整体转换)

6. `settings.json` 加 2 个 key(两侧都加):
   - `preference.locale_name_en_gb = "English (UK)"` / `"English (UK)"`
   - `preference.locale_name_zh_tw = "繁體中文"` / `"繁體中文"`

7. `UiLanguagePicker.tsx` 无需改 — 已经在迭代 `LOCALES`,新 locale 自动出现。

### B. en-GB (PR-29)

聚焦拼写差异,不重写语义:

- color → colour
- customize → customise
- organize → organise
- recognize → recognise
- analyze → analyse
- behavior → behaviour
- center → centre
- favorite → favourite
- dialog → dialogue(部分场景)
- gray → grey
- (其他需逐 key 校对)

技术术语(Whisper / Parakeet / Ollama / OpenRouter / Claude / Groq / BlackHole / Meetily 等)保持英文不变。

注意:`UiLanguagePicker` 中显示的「English (US)」对应「English (UK)」,只改 UI 文案不需改 en-US 内部字串。

## Cross-cutting constraints

1. 不引入新依赖(不引 OpenCC;用静态词表人工转换)。
2. 不动 Rust 后端。
3. 不动 next.config / 任何 build 配置。
4. 每 PR 一个 commit(沿用 Wave 1–6 pattern)。
5. 3 道闸:check:i18n + test:i18n 15-15 + build 11-11。
6. 现有 7 namespace 不变,新 locale 文件结构与现有完全一致。
7. 严格按 `LOCALES` 数组顺序(影响 `UiLanguagePicker` 下拉顺序)。

## Acceptance criteria

- [ ] PR-28 提交后:`LOCALES` 含 4 个 locale;`check-i18n` 跨 4 个 locale 验证通过;`UiLanguagePicker` 下拉出现 4 个选项。
- [ ] PR-29 提交后:en-GB 切换后可见英式拼写(color → colour 等),技术术语保留英文。
- [ ] PR-30 提交后:zh-TW 切换后全部为繁体中文,符合台湾用语习惯,术语与 glossary `approved` 严格对齐。
- [ ] 4 个 locale 切换无缝;en-US / zh-CN 不退化。
- [ ] CI workflow `i18n-check.yml` 仍跑通(无需改,泛化自动生效)。
