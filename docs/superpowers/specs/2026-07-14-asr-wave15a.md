# Wave 15: ASR Chinese Model Evaluation (PR-45a)

> **Base branch:** feature/stability-wave8
> **Series:** Wave 15 (ASR optimization, focused on Chinese meetings)

## Background

Meetily records meetings in Chinese as its primary use case (per the
product brief). The frontend defaults to `large-v3-turbo` for the local
Whisper transcription engine, but the choice was made without systematic
evaluation on Chinese speech. PR-45a delivers that evaluation: a
reproducible benchmark tool, a curated reference table of public results,
and a data-driven selection recommendation.

## Goals

- Provide a self-contained Python tool (`scripts/asr_benchmark/benchmark.py`)
  that measures CER / WER / RTF for any Whisper model variant on
  AISHELL-1 (or similar) test sets.
- Document known public benchmark numbers in `docs/asr_benchmark_zh.md`
  so the team has a shared reference table without re-running.
- Recommend a default model (current `large-v3-turbo` confirmed) and
  provide concrete hardware-driven fallbacks.
- 16 unit tests for the CER / WER / aggregation logic so future changes
  cannot silently break the metric calculations.

## Non-Goals

- **Changing the default model.** PR-45a confirms the existing default;
  changing it would force every user to re-download a multi-GB model.
- **Training a fine-tuned model on Chinese.** Requires GPU resources
  and is tracked separately as PR-45d (post-Wave 15).
- **Hot-word injection / `initial_prompt` tuning.** PR-45c.
- **Hardware-aware auto-selection.** PR-45b (the `getRecommendedModel`
  helper already exists in `lib/whisper.ts` but is not wired into the UI).

## Design

```
whisper.cpp (binary) -> sibling .txt output -> benchmark.py
  parses trans.txt -> per-sample CER / WER
  wall-clock inference time -> per-sample RTF
  aggregated per model -> results.json
```

Key design points:
- **No external Python deps required.** Falls back to a pure-Python
  Levenshtein DP if `python-Levenshtein` is not installed.
- **Stdlib `wave` for duration probing** so no `soundfile` / `librosa`.
- **`-otxt -of` output mode** so the benchmark works with stock
  whisper.cpp without modifying the binary.
- **Configurable sample cap** (`--max-samples`) for smoke runs.

## Sandbox caveat

This environment lacks whisper.cpp binary and a GPU. The benchmark tool
itself is fully tested (CER / WER / aggregation), but the actual
CER numbers in `docs/asr_benchmark_zh.md` are drawn from public sources
(Whisper paper, ggerganov/whisper.cpp issue tracker, community evaluations).
Operators should re-run the tool on their target hardware for ground truth.

## Risks

| Risk | Mitigation |
|---|---|
| Public benchmark numbers drift over time | Cite sources, recommend self-rerun |
| AISHELL-1 doesn't represent real meeting speech | Document caveat; recommend users add their own samples |
| Pure-Python Levenshtein slow on big batches | Suggest `pip install python-Levenshtein` for prod |
| whisper.cpp output format changes | Pin a known-good version in docs |
